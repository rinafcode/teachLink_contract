## Engagement Guidelines

### Be Respectful

- Treat everyone with respect and kindness
- Welcome newcomers warmly
- Be patient with questions
- Use inclusive language
- Respect different experience levels

### Be Helpful

- Share knowledge generously
- Point to documentation when relevant
- Help troubleshoot issues
- Celebrate others' successes
- Mentor new contributors

### Be Constructive

- Focus on solutions, not blame
- Provide actionable feedback
- Acknowledge good work
- Disagree respectfully
- Assume good intentions

### Stay On Topic

- Use appropriate channels
- Keep discussions relevant
- Move extended discussions to threads
- Tag relevant people thoughtfully
- Avoid spam and self-promotion

## Content Creation Guidelines

### Writing Blog Posts

We welcome community blog posts! Guidelines:

1. **Topics**: Technical tutorials, use cases, experiences
2. **Length**: 500-2000 words
3. **Format**: Markdown with code examples
4. **Review**: Submit as PR to blog repository
5. **Credit**: Full attribution to author

### Creating Tutorials

Video and written tutorial guidelines:

1. **Accuracy**: Ensure technical correctness
2. **Currency**: Use latest versions
3. **Accessibility**: Include transcripts/captions
4. **Licensing**: Use CC-BY-4.0 for open sharing
5. **Branding**: Follow brand guidelines

### Social Media Posting

When posting about TeachLink:

âœ… **Do**:
- Use official hashtags: #TeachLink #StellarDev
- Share accurate information
- Tag official accounts
- Credit other contributors

âŒ **Don't**:
- Make price predictions
- Share unverified information
- Speak on behalf of the project
- Share security vulnerabilities publicly

## Event Guidelines

### Hosting Events

Community members can host events:

1. **Propose**: Submit in #events channel
2. **Plan**: Work with community team
3. **Promote**: Announce in official channels
4. **Execute**: Host with community support
5. **Follow-up**: Share recordings/notes

### Event Types

| Type | Description | Support Level |
|------|-------------|---------------|
| Workshops | Hands-on learning | Full support |
| Hackathons | Building projects | Sponsorship available |
| Meetups | Local gatherings | Promotional support |
| AMAs | Q&A sessions | Hosted officially |
| Conferences | Speaking opportunities | Travel grants available |

### Code of Conduct at Events

All events must:
- Follow our [Code of Conduct](../../CODE_OF_CONDUCT.md)
- Have designated moderators
- Provide reporting mechanisms
- Be inclusive and accessible

## Feedback and Suggestions

### How to Give Feedback

1. **GitHub Discussions**: For feature ideas and project feedback
2. **Discord #ideas**: For quick suggestions
3. **Surveys**: Participate in community surveys
4. **AMAs**: Ask during Q&A sessions

### Feedback Guidelines

- Be specific and actionable
- Explain the "why" behind suggestions
- Consider multiple perspectives
- Be open to discussion
- Accept that not all feedback can be implemented

---

## Conflict Resolution

### If You Witness Issues

1. Don't engage publicly
2. Report to moderators
3. Provide context and evidence
4. Let moderators handle it

### If You Experience Issues

1. Document the incident
2. Report to moderators privately
3. Don't retaliate
4. Await response

### Reporting

- **Discord**: DM a moderator
- **GitHub**: Contact maintainers
- **Email**: conduct@teachlink.io
